动态规划用于解决多阶段决策最优化问题
## 动态规划的核心思想
通俗的讲dp的核心就是记住已经解决过子问题的解，通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。dp常常适用于有重叠子问题和最优子结构性质的问题，动态规划方法所消耗的时间往往远小于朴素解法。
## 动态规划可解问题的判断标准（<font color="red">必须满足最优化原理和无后效性，是否为重叠子问题都可，但重叠子问题能最大化利用动态规划的特点</font>）
- 最优化原理：如果问题的最优解所包含的子问题的解也是最优的，就称该问题具有最优子结构，即满足最优化原理
- 无后效性：即某阶段状态一旦确定，就不受这个状态以后决策的影响。也就是说，某状态以后的过程不会影响以前的状态，只与当前状态有关。
- 有重叠子问题：即子问题之间是不独立的，一个子问题在下一阶段决策中可能被多次使用到。（该性质并不是动态规划适用的必要条件，但是如果没有这条性质，动态规划算法同其他算法相比就不具备优势）

满足最优化和无后效性即可解答，但若问题不满足有重叠子问题，就无法复用前一个状态的计算结果用于后一个状态的计算，这时动态规划的优势便不复存在，可能并非最优解。

分治和动态规划的区别
共同点：两者都要求原问题具有最优子结构性质，都是将原问题分而治之，分解成若干个规模较小的子问题，然后将子问题的解合并，最终得到答案。
不同点：分治法将分解后的子问题看成相互独立的，通常用递归来做。动态规划将分解后的子问题理解为相互间有联系，有重叠部分，需要记忆，通常用迭代来做。
## 动态规划可解问题的常见特征
求方案数问题
1. 多少种方式从左上角移动到右下角
2. 多少种方法选出k个数使得和是sum

最优解问题
1. 从左上角到右下角路径的最大数字和
2. 最长上升子序列长度

可行性问题
1. 找零钱，是否能找开
2. 能不能选出k个数使得和是sum

## 动态规划解题四部曲
1. 确定状态（确定dp数组及其下标的含义，研究最优策略最后一步，确认子问题）
2. 确定状态转移方程（即前一个状态和后一个状态间的递推公式，根据子问题定义可以直接得到）
3. 确定初始条件和边界情况
4. 确定dp数组计算顺序（一般是从小到大，因为后面计算要用前面结算结果加速，这是动态规划的精华所在，就是记忆化搜索，相比于递归）

经过上面四个步骤动态规划的代码就写完了
第二步就是举出一些例子进行验算
第三步分析时空复杂度

求状态表示的小技巧：一般由题目要求什么，状态表示就表示成什么。

### 动态规划常见优化思路
1. 滚动数组优化空间复杂度
2. 特定问题优化，求某一个数组中除去第i个元素之外的最小值
		1. 求前缀最小值数组和后缀最小值数组
		2. 只需要遍历一次获得最小值和次小值即可
		3. 第二种方案更好

### 斐波那契数

- 题源：[509. 斐波那契数](https://leetcode.cn/problems/fibonacci-number/)

```
class Solution:
    def fib(self, n: int) -> int:
        if n < 2:
            return n
        f0, f1 = 0, 1
        for i in range(2, n + 1):
            fn = f0 + f1
            f0, f1 = f1, fn
        return fn
```

### 不同路径

- 题源：[LeetCode 62 不同路径](https://leetcode.cn/problems/unique-paths/)
- 难度：M

无状态序列型动态规划

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # dp[i][j] 表示机器人从 (0, 0) 到 (i, j) 的不同路径数量
        # dp[i][j] = dp[i][j-1] + dp[i-1][j] 其中 j-1 >= 0 i-1 >= 0
        dp = [[1] * n for _ in range(m)]
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = dp[i][j - 1] + dp[i - 1][j]
        return dp[m - 1][n - 1]
```

## 划分型动态规划

解题思路：通常是在序列型动态规划的基础上，增加一个维度存储段数信息，f[i][j] 表示前 i 个元素分成 j 段有某性质
- 如果题目不指定段数，用f[i]表示前i个元素分段后的可行性/最值，可行
性，方式数： Perfect Squares, Palindrome Partition II
• 如果题目指定段数，用f[i][j]表示前i个元素分成j段后的可行性/最值，可
行性，方式数：Copy Books
