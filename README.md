对于不了解的东西，我的学习方法是这样的:
1.先不问为什么，会用，知道这个东西能解决什么问题，在什么场景下用更为合适。在公司中，先让自己成为一个比较好用的工具人。
2.通过反复实践，在实践中去思考:做这件事是为了解决什么问题？分几步走？每一步的作用是什么？最终要的效果是什么？它有什么缺点与优点？如何去改进、优化？
3.经历的多了，如果仔细去总结对比，总会发现一些共性的东西，把它抽象出来，最终总结出一套通用的解决问题的方案。这个时候再搭配一些理论方面的书籍，绝对可以理解的更为深刻。
4.以一个设计者的角度去看问题:假如换做是你，你能不能解决问题？能不能把问题解决的更好？要尝试打破以前那些理论的条条框框，找到问题的本质，采用自己独特的方法去更高效地解决问题。

在不使用同步机制的情况下，A、B线程各自执行i++一百次，假设初始i==0，则执行完毕后，i的取值范围可能是2-200

A、B线程执行一次，此时A线程CPU1缓存i==1，B线程CPU2缓存i==1，内存i==0

A线程正常执行99次，此时CPU1缓存i==99，内存i==99

B线程此时CPU2缓存写回内存，内存i==99被覆盖为i==1

A线程执行第100次，CPU1缓存i==2，内存i==1

B线程执行到100次，CPU2缓存i=100，内存i==100

A线程此时CPU1写回内存，内存i==100被覆盖为i==2



正常执行值为200





java四舍五入

```java
double num = 3.1415926;
String result = String.format("%.4f", num);
System.out.println(result);//输出结果为：3.1416
//%.2f 中 %. 表示 小数点前任意位数 2 表示两位小数 格式后的结果为f 表示浮点型。
```

```java
double num = Math.round(5.2544555 * 100) * 0.01d;
System.out.println(num);
//输出结果为：5.25
//最后乘积的0.01d表示小数点后保留的位数(四舍五入)，0.0001 为小数点后保留4位，以此类推…
```
