Java 中为什么重写 equals 方法同时一定要重写 hashCode 方法呢？


String 类中 equals 方法实现：先比较两个字符串对象内存地址，相同就返回 true，不同就比较两个字符串的值，值相同就返回 true，不相同返回 false。


Java 中每个对象都会有 hashcode 方法，hashcode 值用于标识一个对象，不同对象 hashcode 可能会相同（哈希冲突），相同对象 hashcode 值一定相同



equals 和 hashcode 都用来分析两个对象是否相同，但不同的是 hashcode 专为 hash 设计，当对象存储散列表时，需要以 hashcode 标识该元素以进行定位

如果重写 equals 方法后，不重写 hashcode 方法，就有可能导致两个对象被 equals 判断为相同，但 hashcode 却不同的问题，这样就导致，A与B对象 equals 判断为真，且A已存入 Map，但通过B的hashcode却从Map中获取不到对象的问题，这与实际使用割裂开来，两个都重写是为了保证 equals 与 hashcode 对函数鉴别方法一致
